# airflow-clickhouse-etl-pipeline

# ETL пайплайн на Airflow + ClickHouse (дефектура СТМ)

В данном репозитории представлен пример небольшого личного production ETL пайплайна для обработки данных закупочных цен и расчёта витрины.

Код обезличен.

## Описание

Пайплайн реализует полный цикл обработки данных из внешнего источника:

- загрузка данных из Excel-файла
- трансформация и преобразование
- загрузка в ClickHouse
- построение витрины с бизнес-метриками
- инкрементальное обновление данных

## Архитектура

Общий поток данных: Excel → Airflow → ClickHouse → аналитическая витрина

##  Логика работы DAG

- Проверка наличия нового файла в директории
- Если файл найден:
  - парсинг и загрузка в ClickHouse
  - перемещение файла в архив
- Если файл отсутствует:
  - пропуск загрузки
- В обоих случаях:
  - выполняется инкрементальный пересчёт витрины
## Инкрементальная обработка

Пересчёт выполняется не по всей истории, а только по последнему окну равному 3-м дням
## Особенности реализации

- Используется `data_interval_end` из контекста для управления датами (вместо `today()`) — для идемпотентности и возможности перезапуска DAG за прошлые даты
- Реализована идемпотентная загрузка (DELETE + INSERT) — исключает дубли при повторных запусках
- Используется ветвление DAG (`BranchPythonOperator`) — обработка сценариев с файлом и без файла
- Применён ASOF JOIN в ClickHouse — корректная подстановка актуальных цен при нерегулярных загрузках
- Автоматическая архивация обработанных файлов с добавлением таймстампа — для повторного анализа

##  Обработка файлов

- Выбирается последний загруженный файл в директории
- После успешной загрузки файл перемещается в архив
- К имени файла добавляется дата и время обработки
- Исключается повторная обработка одного и того же файла в рамках одного запуска
